(우아한테크코스 6기) 프리코스를 마무리하고.

2023-11-20   |   블로그

지난 10월부터 경험한 우아한 테크코스 6기 지원과 프리코스 5주의 경험을 남깁니다.

우아한테크코스는 '몰입'의 과정이다.

우아한테크코스는 개발자로 전직하는 것에 대한 관심이 생겼을 때부터 인지를 하고 있었다. 배달의민족을 서비스하는 '우아한형제들'에서 만든 '문제를 해결할 수 있는' 개발자를 양성하는 코스로 알고 있었다. 퇴사를 하고, 다음 커리어를 고민하며 쉬고 있을때 우아한테크코스 6기 모집 공고를 확인할 수 있었다. (우아한테크코스,테코블, 테코톡 페이지는 전 직장 업무상 북마크를 해뒀었었다.)

우아한테크코스(이하 우테코)는 지원 단계에서 '넌 어떤 개발자가 되고 싶은거야?'와 '개발자가 되는 과정에서 몰입 잘 할 수 있겠어?'라는 질문을 받게된다. 이 질문들은 정말 많은 생각을 요구했다. 내가 생각하는/되고싶은 개발자의 모습은 어떤 모습일까를 다시 천천히 고민해봐야 했다. 이전에 내가 해본 몰입의 경험은 어떤 것이 있는지도 노트 한 장을 가득 적어가면서 고민했다. 이 고민을 하던 중에 우테코 팀에서 라이브 입학 설명회를 하셨던 것 같다. 라이브 입학 설명회가 확실히 고민의 범위를 잘 좁혀줬던 것 같다.

우테코의 지원 과정 역시 나의 생각과 경험들을 다시 조립해보는 '몰입'의 과정이었던 것 같다. 질문 문항마다 정리된 내 생각들을 솔직하게 작성했다. 작성한 지원서의 맞춤법 검사를 하는 것을 끝으로 우테코 프리코스 과정을 시작할 수 있었다.

프리코스는 '설렘'과 '몰입'의 달리기다.

프리코스는 4주간 GitHub에 공개된 과제를 스스로 해결해보고 정해진 방식대로 커밋 > PR하여 제출하는 방식으로 진행되었다. 제출이 완료되면 공통 피드백을 제공해주셨다. 모든 과정이 메일을 통해 안내되었다. 메일 내용을 꼼꼼하게 잘 확인해야 했다. 매주 수요일 오후 3시에 새로운 미션이 제시됐다. 제시된 미션은 다음 화요일 자정까지 정해진 방법으로 정확하게 제출해야 한다. 정확하게 한 주 동안 주어진 요구사항을 충족하는 코드를 작성하면 되었다. 요구사항을 분석하고 코드를 작성하기에는 충분한 시간이라고 생각했다.


1주차는 숫자 야구 게임 ⚾️을 구현하는 미션이 주어졌다. 첫 미션이었기에, 나는 주어진 요구사항을 잘 이해하는 것에 초점을 맞췄다. 미리 준비된 라이브러리도 잘 이해해야 했다. 요구사항과 주어진 라이브러리를 이용해서 동작하는 코드를 만들기 위한 구현 과정을 문서로 미리 기록해야 하는 조건이 있었다. 나는 아이패드로 앱이 어떻게 유기적으로 동작해야 할지 플로우를 그려봤다. 그리고 나서 내가 구현해야 할 기능의 목록을 README에 작성한 다음 본격적으로 코드를 작성했다.

1주차 미션을 수행하면서 App.js 파일 하나에 거의 모든 코드를 작성했다. 중간에 요구사항에 적혀있던 '상수 값을 구분하기'를 확인하면서 상수 값 구분용 파일을 생성하긴 했다. App 클래스 내부 메서드 하나가 값을 입력 받아 특정 연산을 하고 계산된 값을 반환하는 전체 로직을 하나의 메서드에서 처리하는 식으로 코드를 작성해, 너무 많은 일을 하게 만들었다. 단순히 다른 API를 감싸기만 한 불필요한 메서드를 만들기도 했다.

1주차 미션 코드를 PR하고, 플랫폼에서 제출을 완료하고 나서 다른 지원자분들의 코드를 한 번씩 살펴봤다. 역할에 맞게 코드를 잘 분리한 분도 있었고, 작은 앱이라도 구조를 탄탄하게 짠 코드도 있었다. 역시 주변에 함께 성장할 수 있는 좋은 동력원들이 많을 것이라는 내 생각이 맞았다. 그 때 우테코에 지원한 것에 한 번 더 설렜던 것 같다. 우테코에 최종적으로 탑승하게 되면 내가 여러 거목들 사이를 걸을 수 있겠구나 싶었다.

1주차 미션이 종료되고 공통 피드백이 주어졌다. 제공되는 공통 피드백과 내가 제출한 코드를 비교해보면서 나의 부족한 부분을 다시 고민해볼 수 있다는 것이 프리코스의 큰 장점 같았다. git commit 메시지에 대한 피드백이 특히 1주차 공통 피드백에서 많은 도움이 되었다.


2주차는 자동차 경주 🏎️ 미션이 주어졌다. 1주차와 비슷한 요구사항이 있었다. 추가 요구사항도 존재했다. Jest를 이용한 테스트 코드를 직접 작성하는 과정도 필요했다. 테스트 코드를 직접 작성하라니! 우테코스러운 요구사항이라고 생각했다. 테스트 코드 작성에 참고할만한 예시 파일도 제공되었다.

1주차에 스스로 회고한 부분을 2주차 코드에 반영하려고 노력했다. 메서드의 역할을 분리시켰다. 메서드의 depth가 2를 넘지 않아야 한다는 추가 요구사항도 있었기 때문에 역할을 분리시키지 않을 수 없었다. 아쉬웠던 점은 여전히 App.js에 대다수의 코드가 몰려 있었던 것이다. 또한 1주차 처럼 단순히 콜백을 리턴하는 불필요한 함수를 여전히 만들었던 것도 아쉽다.

1주차에 비해 좋았던 점은 테스트 코드를 작성하는 것이었다. 미션을 진행하는 한 주동안 테스트 코드를 고민하는 것에 대부분의 시간을 보냈다. 실제로 테스트 코드를 작성하면서 이전에 작성했던 로직의 오류를 발견하기도 했다. '이래서 테스트 코드를 작성하는 구나'하고 직접 경험했다. 함수의 역할을 구분하니까 특정 함수를 테스트 하기 위해 내부에서 실행되는 함수의 결과 값을 mocking 하는 방법을 찾는데 많은 검색이 필요햇던 것 같다. 가장 좋았던 것은 이 과정이 정말 즐거웠다는 점이다!


3주차는 로또 🎟️ 미션이었다. 1, 2주차와 비슷하게 입력받은 값으로 특정 연산을 하여 동작하는 코드를 작성하는 미션이었다. 2주차와 마찬가지로 추가 요구사항이 있었다. 2주차 공통 피드백과 3주차 추가 요구사항을 기반해서 이번에는 확실히 View, Domain, App을 구분해서 코드를 작성했다. 2주간의 미션과 피드백으로 코드를 분리하고 역할을 제한시키는 것이 확실히 손에 익었다는 생각이 들었다.

3주차의 아쉬운 점은 Domain 클래스에 있었다. 이건 3주차 피드백을 받고 '아 문제였구나'라고 인지했던 부분이다. Domain 클래스에서 변수를 위한 변수가 제법 있었던 것 같다. 즉, 이건 연산을 통해서 계산될 수 있는 값도 멤버 변수로 불필요하게 선언했다는 점이다. 또한 불필요한 getter 메서드도 제법 있었던 것 같다. 이건 테스트 코드를 작성하기 위한 코드였던거 같다. 한마디로 정말 불필요한 코드였다. 공통 피드백에서 '객체가 일하게 만든다.'라는 피드백과 나의 아쉬움을 정확하게 짚은 '테스트를 위한 코드는 구현 코드에서 분리되어야 한다.' 라는 피드백에 뼈가 아팠다.

3주차의 좋았던 점은 1, 2주차에 하지 못했던 코드 역할 분리다. 요구사항이 늘어가면서 Domain 클래스가 무거워지긴 했지만, View를 담당하는 코드를 작성하면서 코드를 더 명확하게 짤 수 있었던 것 같다. 그리고 테스트 코드를 작성하는 것에 자신감이 조금 붙은 점도 역시 좋았다. 우테코를 통해서 '성장'하는 재미를 붙일 수 있겠다는 확신이 이 시점에 생겼던 것 같다.


4주차 미션은 크리스마스 🎄였다. 1 ~ 3주차와는 미션 진행 방식이 달랐다. 변경된 진행 방식을 놓치지 않기 위해서 제시된 방법을 철저하게 따르는 것에 집중했다. 미션에서 요구하는 바는 1 ~ 3주차와 결이 거의 비슷했다. 3주차 미션을 제출하고 나서, 나는 다른 지원자들의 코드를 보며 어떤 식으로 내 코드 구조를 설계할 수 있을지에 대한 레퍼런스를 많이 찾았봤다. 나와 전혀 다른 방식으로 오류 값을 검증하고, 앱을 다시 작동시키는 코드들이 많았다. 대다수가 try ~ catch 구문을 많이 사용하셨었다. 그래서 나도 4주차 미션에는 해당 구문을 사용해서 반복 작업을 걸고, 오류를 검증하기로 했다.

3주차까지의 학습이 4주차 미션을 수행하는 것에 정말 큰 도움을 줬다. 1 ~ 3주차의 경험치가 없었다면 4주차의 미션이 아득했을 것 같았다. 이래서 사람은 성장하기 위해서 꾸준하게 반복해야 하는 것 같다. 그리고 우테코에서 강조하는 '몰입'도 정말 중요했던 것 같다. 4주차에는 미션을 진행하는 것이 전혀 부담되지 않았다. 코드를 작성하는 과정이 즐겁기만 했다. 몰입해서 느낄 수 있는 즐거움이었다.

4주차 코드는 일부러 빨래방을 찾아가서 많이 작성했다. 이상할 수 있는데, 나는 세탁기가 돌아가고 스팀향이 나는 공간에서 집중이 정말 잘된다. 이것도 우테코를 진행하면서 발견한 사실 중 하나다. :)

4주차 미션 제출 시간이 지나고 나니까 다음 주 부터 미션이 메일로 오지 않는다는 사실이 아쉽다는 감정이 생겼다. '내가 4주간 정말 잘 몰입했구나'하고 생각하면서 프리코스 일정을 마무리했다.

그래서 나는.

프리코스를 마무리하고 나니까 우테코의 본 코스에 더 참여하고 싶다는 그 욕망(?), 간절함이 강해졌다. 주어진 요구사항으로 문제를 해결할 수 있는 코드를 작성해가는 그 과정 자체가 너무 즐거웠기 때문이다. 우테코의 본 과정에서는 그 과정을 더 즐겁게 반복할 수 있겠다 싶었다. 앞서 말했던 것처럼 나보다 한.. 세 뼘은 더 높을 것 같은 거목들 사이에서 말이다.

우테코 프리코스 덕분에 정말 즐거운 4주를 보낼 수 있었던 것 같다. 12월에 최종 테스트 진행자에 대한 안내 메일을 보내주신다고 하는데 꼭 테스트 대상에 포함되면 좋을 것 같다. 😎 메일로 결과를 확인하고 더 업데이트 할 내용들이 있다면 이 블로그에 이어쓸 생각이다. 아자잣
